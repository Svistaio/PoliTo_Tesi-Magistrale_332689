    \section{Codice}\label{apnCodice}

        Il codice è reperibile in questo repositorio \cite{Taralli2026} ed è stato scritto interamente in Python 3.13.7 mentre le simulazioni sono state eseguite su una macchina con sistema operativo Windows e processore Intel Core i5-7500 (quadri-nucleo, 3.40 GHz).
        
        %\begingroup «main.py»
            Innanzitutto \texttt{main.py} nel \cref{lstMain} è il \textit{file} principale contenente quasi tutta la logica del programma:
    
            \begin{itemize}[
                label=\(\diamond\),
                %topsep=0.5em,
                %parsep=0em,
                %itemsep=0.25em,
                %leftmargin=2em,
                %rightmargin=1.5em,
                % \leftmargin + \itemindent = \labelindent + \labelwidth + \labelsep
                %itemindent=!,
                %labelindent=30pt,
                %labelwidth=!,
                %labelsep=!,
            ]
                \item \texttt{clsGUI.GatherParameters()} raccoglie tutt'i parametri attraverso un'interfaccia grafica mostrata nella \cref{figInterfacciaGraficaParametri};
                \item \texttt{libD.ExtractRegionData()} estrae e analizza i dati ISTAT \cite{ISTATLimiti1991,ISTATCensimento1991,ISTATPendolarismo1991};
                \item \texttt{libD.LoadRegionData()} carica i dati appena o già estratti;
                \item \texttt{libN.NetworkAnalysis()} è la classe contenente le funzioni per riprodurre i risultati di \cite{DeMontis2007};
                \item \texttt{libK.KineticSimulation()} è la classe contenente le funzioni per simulare l'\cref{algMonteCarlo} e trattare i risultati;
                \item \texttt{libK.ParametricStudy()} è la classe chiamata in alternativa alla precedente se si è interessati a uno studio parametrico\footnotemark{}.%
                \footnotetext{Di fatto non è altro che \texttt{libK.KineticSimulation()} tante volte quanti sono le variazioni del parametro studiato.}
            \end{itemize}
